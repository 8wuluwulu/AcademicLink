# Проект AcademicLink

**AcademicLink** — это комплексная система (CRM) для репетиторов, предназначенная для управления записями студентов. Система объединяет веб-интерфейс (API) для приема заявок и Telegram-бота для административного управления.

## Основной функционал

### 1. Веб-интерфейс (FastAPI)
- Прием заявок на обучение через REST API.
- Валидация данных студентов (имя, телефон, тип услуги, время).
- Автоматическое уведомление репетитора в Telegram о новых записях.

### 2. Telegram-бот (Aiogram 3)
- **Дашборд репетитора:** Сводка на сегодня, количество новых заявок и общее число учеников.
- **Управление расписанием:** Просмотр записей с группировкой по датам, подтверждение или отмена занятий (с указанием причины).
- **База учеников:** Поиск по номеру телефона, просмотр истории занятий каждого ученика, быстрый переход в чат Telegram.
- **Настройки профиля:** Возможность временно приостановить прием заявок (режим "Пауза").

### 3. База данных (PostgreSQL + SQLModel)
- **Студенты:** Хранение контактных данных и связей с Telegram.
- **Репетиторы:** Управление профилями и статусом активности.
- **Записи (Bookings):** Полный жизненный цикл занятия (Ожидание, Подтверждено, Отменено).

---

## Технологический стек

- **Язык:** Python 3.12+
- **Web Framework:** [FastAPI](https://fastapi.tiangolo.com/) (асинхронный сервер)
- **Bot Framework:** [Aiogram 3.x](https://docs.aiogram.dev/)
- **ORM:** [SQLModel](https://sqlmodel.tiangolo.com/) (объединяет SQLAlchemy и Pydantic)
- **Database:** PostgreSQL
- **Migrations:** Alembic
- **Config Management:** Pydantic Settings (использование `.env` файлов)

## Как это работает (Workflow)

1. Студент оставляет заявку на сайте (через API).
2. Система создает запись в БД и отправляет уведомление репетитору в Telegram.
3. Репетитор через бота подтверждает запись или отменяет её.
4. В боте репетитор всегда видит свое актуальное расписание и может быстро связаться с любым учеником.
